# 公开 Prompt 原则详细总结与可复用模板（适配 LangGraph/3D 打印/Coding Agent）

## 一、官方权威 Prompt 原则（厂商标准，直接套用）

### 1. OpenAI 官方六大核心原则（2023 年 12 月发布）

| 原则               | 核心要点                         | 实操技巧                                                                                  |
| ------------------ | -------------------------------- | ----------------------------------------------------------------------------------------- |
| **清晰指令**       | 明确任务边界、输出格式、评估标准 | 使用分隔符（```/###）隔离指令与上下文；用“必须”“禁止”等强约束词；避免模糊表述如“帮我处理” |
| **提供参考文本**   | 减少幻觉，提高准确性             | 提供行业标准、设备手册、代码文档等参考资料；标注信息来源与时间范围                        |
| **拆分复杂任务**   | 降低理解难度，提高执行精度       | 按“总目标 → 子任务 → 输出要求”分层；适配 LangGraph 多节点协作                             |
| **给模型思考时间** | 引导逐步推理，避免仓促结论       | 使用思维链（CoT）：“让我们一步步思考”；要求标注推理步骤与依据                             |
| **使用外部工具**   | 拓展能力边界，增强可验证性       | 指定可用工具（如搜索、代码执行、3D 模型分析）；明确工具调用条件与参数                     |
| **系统测试变更**   | 迭代优化提示词，确保效果稳定     | 控制单一变量测试；记录每次调整的输出变化；建立评估指标（如准确率、效率）                  |

**关键技巧示例**：

```
### 指令 ###
你是一名3D打印前处理专家，必须按照以下步骤处理牙模文件：
1. 检查STL文件完整性（无漏洞、无重复面）
2. 优化支撑结构（咬合面禁止支撑，支撑密度0.2-0.3）
3. 控制文件体积≤1.0GB（设备最大支持）

### 参考文本 ###
牙模3D打印行业标准：分辨率80-120 DPI，支撑接触面积<5%，打印精度±0.05mm

### 输出要求 ###
JSON格式，包含：{"file_check": "结果", "support_params": {"密度": 0.25}, "file_size": 0.8GB}
```

### 2. Anthropic Claude 提示工程原则

- **角色先行**：开篇定义 AI 身份、专业领域与能力边界，避免越界输出
- **分步指令**：复杂任务拆分为“先做 A→ 再做 B→ 最后做 C”的线性流程
- **输出格式强制**：明确指定 JSON/Markdown/代码等格式，使用“输出必须为 XX 格式”强约束
- **分隔符隔离**：用###分隔角色、指令、示例、上下文，提高解析效率
- **幻觉防控**：要求标注信息来源；提供反例；加入“无法确认时必须说明”条款
- **示例驱动**：提供 1-3 个输入输出示例，尤其适合非标任务

### 3. Google Gemini 提示指南

- **明确目标**：区分“信息查询”“创意生成”“任务执行”等不同目标类型
- **上下文完整**：提供场景背景、用户偏好、行业标准等必要信息
- **多模态提示**：标注输入类型（文本/图像/3D 模型）；明确跨模态转换要求
- **输出引导**：用“以 XX 开头”“包含 XX 字段”等引导输出结构
- **错误处理**：预设容错逻辑（如参数异常时返回默认值；设备连接失败时重试 3 次）

### 2. IBM 三大核心提示原则

- **Specificity（明确详细）**：用具体数值、可量化指标代替模糊描述
  - 错误：“优化 3D 打印参数”
  - 正确：“将牙模分辨率从 120 降至 100，支撑密度保持 0.28，确保文件体积 ≤1.0GB”
- **Step-by-step structuring（分步拆解）**：按任务执行顺序设计提示
- **Iterative refinement（迭代优化）**：建立“执行 → 评估 → 优化”闭环，适配 Agent Reflection 架构

### 3. LangChain 提示模板设计原则

- **模块化**：用 PromptTemplate 封装固定指令，通过变量注入动态上下文
- **可复用**：为同类任务创建模板库（如 3D 打印前处理模板、代码生成模板）
- **状态关联**：明确读取/更新的状态字段（如`state["execution_error"]`→`state["reflection_result"]`）
- **上下文注入**：自动填充设备参数、行业标准、历史执行记录等上下文
- **输出验证**：要求输出包含可验证字段（如反思结果的`root_cause`），便于后续节点评估

## 三、垂直场景专项 Prompt 原则（适配核心应用）

### 1. LangGraph 节点提示原则

- **单一职责**：每个节点提示仅聚焦核心功能（执行器=执行步骤，反思器=评估标准）
- **状态管理**：明确节点可访问的状态键（如`state["model_params"]`）和需要更新的键（如`state["processed_file"]`）
- **循环安全**：限制最大重试次数（如 3 次），防止死循环；设置退出条件（如任务成功或无法修复）
- **错误处理**：预设异常情况的处理逻辑（如设备连接失败时返回错误码并触发重试）

### 2. 3D 打印自动化 Agent 提示原则

- **行业适配**：根据牙模/工业零件/轮胎等领域提供对应的工艺标准
- **设备协同**：明确设备参数（如 SN、IP、最大文件体积）与通信协议
- **质量优先**：强调关键区域精度要求（如牙模咬合面、工业零件壁厚）
- **效率平衡**：在保证质量的前提下优化处理速度（如批量处理分盘逻辑）

### 3. Coding Agent 提示原则

- **功能与性能双约束**：不仅要求实现功能，还需指定时间复杂度、空间复杂度、代码规范
- **上下文完整**：提供项目依赖、现有代码结构、接口定义、测试要求
- **输出标准**：代码+测试用例+注释+性能分析；用分隔符隔离代码与说明
- **可维护性**：要求代码符合 PEP 8/Google Style 等规范；加入文档字符串与类型提示

## 四、可复用 Prompt 模板库（直接复制使用）

### 模板 1：LangGraph 反思器节点提示模板（适配 Agent Reflection 架构）

```
## Role ##
你是LangGraph 3D打印Agent的反思器节点，负责评估前处理流程的执行结果，定位问题根因并生成可执行的优化策略。你的专业领域包括牙模、工业零件的前处理参数优化与设备协同逻辑。

## Context ##
- 行业标准：牙模分辨率80-120 DPI，支撑密度0.2-0.3，文件体积≤1.0GB
- 设备参数：联泰ONE-PD打印机，最大支持文件体积1.0GB，打印精度±0.05mm
- 历史执行记录：{state["execution_history"]}

## Task ##
1. 对比实际输出({state["execution_result"]})与行业标准/设备参数，判断执行状态（成功/失败）
2. 若失败，定位根因（如分辨率过高、支撑密度不合理、文件体积超标）
3. 生成针对根因的参数调整建议（需具体数值）与流程优化步骤
4. 评估优化建议的可行性（必须符合设备约束与行业标准）

## Output Requirements ##
JSON格式，包含以下必填字段：
- execution_status: "success"或"failure"
- root_cause: 字符串数组，列出所有问题根源
- param_suggestions: 字典，包含具体参数调整（如{"resolution": 100, "support_density": 0.25}）
- process_suggestions: 字符串数组，列出流程优化步骤
- feasibility: "high"/"medium"/"low"，评估建议可行性

## Constraints ##
- 禁止建议降低牙模核心区域（咬合面）的分辨率
- 优化策略必须兼顾效率与质量，不能仅追求文件体积达标
- 所有参数调整必须提供依据（如"分辨率从120降至100，基于设备最大文件体积1.0GB的约束"）
```

### 模板 2：3D 打印前处理执行器节点提示模板

```
## Role ##
你是3D打印前处理执行器，负责牙模STL文件的自动化处理，使用联泰ONE-PD插件API完成操作。

## Input ##
- 模型文件：{model_file_path}
- 初始参数：{initial_params}（包含分辨率、支撑密度、壁厚等）
- 设备信息：{device_info}（包含SN、IP、最大文件体积）

## Task Steps ##
1. **文件检查**：验证STL文件完整性（无漏洞、无重复面、无反向法线）
2. **支撑优化**：
   - 咬合面禁止添加支撑
   - 支撑密度控制在0.2-0.3之间
   - 支撑接触面积<5%
3. **分辨率调整**：根据文件体积动态调整（目标≤1.0GB）
4. **切片处理**：使用联泰ONE-PD插件生成打印文件
5. **参数校验**：确保所有参数符合设备约束与行业标准

## Output ##
JSON格式，包含：
- file_check_result: "pass"或"fail"，及详细说明
- final_params: 最终处理参数（分辨率、支撑密度等）
- file_size: 处理后文件体积（单位：GB）
- sliced_file_path: 切片文件路径
- execution_status: "success"或"failure"

## Error Handling ##
1. 文件检查失败：返回错误码E101，说明具体问题（如"存在3处漏洞"）
2. 参数调整后仍超标：返回错误码E102，建议人工干预
3. 设备连接失败：重试3次，每次间隔5秒；仍失败返回错误码E103
```
